# World Model V2 core architecture: boundaries, residual world, bounded cognition, hybrid mechanisms, honest truncation

This document describes the default-on core architecture that governs **every** World Model V2
simulation on the production route (`swm.facade.forecast(architecture="world_model_v2")` →
`unified_runtime.simulate_world` → `structural_runtime.simulate_structural_ensemble`):

```
question, context, evidence, intervention, as-of time, and horizon
→ independently generated structural world models          (ensemble_compiler)
→ explicit boundary for each structural model              (world_boundary.py)
→ included actors, populations, institutions, mechanisms, and external systems
→ residual outside-world process for what remains beyond the boundary   (outside_world.py)
→ heterogeneous world particles                            (joint hypotheses × memory × family)
→ actor-local observation → attention → finite working memory → imperfect persistent memory
→ situated interpretation → limited action search → one actor choice    (bounded_cognition.py)
→ validated human or nonhuman mechanism execution          (mechanism_spec.py, kernel_ports.py)
→ dynamic discovery and promotion of newly relevant actors and mechanisms (boundary_monitor.py)
→ event-driven temporal rollout                            (temporal_runtime.py)
→ terminal result, truncation, or under-modeled classification          (result.py, truncation.py)
```

## Two mandated statements

**LLM actors are not treated as actual independent human minds.** World Model V2 creates
independently situated actor instances with persistent heterogeneous cognition and, where
available, diversified model families. When only one model family is configured, every result
reports `model_family_monoculture = true`.

**Compute exhaustion never changes an actor's behavioral model.** The affected branch stops and
is reported as truncated. Budget exhaustion, provider failure, parse failure, long context,
deep cascades and timeouts never replace a qualitative LLM actor with `ActorPolicyModel`,
random utility, a logit, a probability table, a persona score, a population average, a direct
LLM probability, or a hardcoded default action.

This architecture does **not** claim universal causal coverage. Missing high-sensitivity
components classify results honestly instead of completing them apparently.

## Explicit world boundaries (`world_boundary.py`)

Every structural model gets its own typed `WorldBoundary`, generated by an actual LLM call and
cross-checked against the executable plan (never inferred silently from `plan.entities`). It
answers: what is simulated individually; what is a population/aggregate; what is a nonhuman
system; what stays outside the detailed world; why each inclusion/exclusion was made; what
evidence supports it; which omitted component could still change the answer; and what triggers
expansion. Three independent critics attack it (omitted-actor, omitted-system,
boundary-adversary), then bounded adversarial sensitivity analysis tests the highest-sensitivity
omissions with qualitative labels only — no minted probabilities. Unresolved decisive components
classify the run `under_modeled_boundary`, never a limitations bullet.

## Residual outside-world process (`outside_world.py`)

The boundary does not make the outside world vanish. Each model carries an
`OutsideWorldProcess`: scenario-identified external event families (news, competitor actions,
weather, outages, legal changes … — identified per scenario, never a fixed global ontology) with
arrival models that must be *defensible* (observed base rates, grounded scenario data, fitted
hazards, documented broad priors, exact schedules) or explicitly `unresolved` — an LLM never
mints a precise event probability. Unresolved families are never sampled; they surface as
unresolved external risks and can classify `under_modeled_external_process`. Arrivals enter the
shared world **only** through typed entry mechanisms (observation delivery, resource change,
price change, capacity change, institutional rule change, physical interruption, population
exposure, new available action, actor promotion) on the canonical clock and queue — never by
writing the answer, terminal utility, a recommendation rank, or an actor's reaction. Impacts
whose full mechanical effect is not yet representable are recorded as downgraded entries, not
invented.

## Dynamic promotion (`boundary_monitor.py`)

The boundary is expandable during rollout. The boundary monitor watches semantic and outside
events for the boundary's own promotion rules; promotion happens at the exact event time that
made the component relevant. A promoted actor receives only reconstructable actor-local history
(public-visibility items up to now — never branch-private state) and initializes cognition
lazily at their first real decision trigger. A promoted population member is recorded as a
segment decrement so aggregates never double-count them. External systems can be promoted from
residual to internal handling, recorded honestly.

## Bounded cognition (`bounded_cognition.py`)

Actor cognition is a mechanical staged pipeline, not a coefficient catalog and not one enormous
prompt: available observations → attention selection (temporal/channel rules + LLM judgment;
missed items recorded with reasons; persistent-channel misses re-queue) → finite, stateful
working memory (capacity from the actor's qualitative situation, with a hard *implementation*
safety maximum that is not presented as psychology; never "last N global events") → cue-based
retrieval over imperfect persistent memory (`ActorMemoryState`: episodic memories with source
traces and append-only reinterpretations, beliefs that may contradict each other and persist
un-averaged, commitments, habits, unresolved tasks; retrieval can fail) → situated
interpretation (actor-local; leakage-screened; may activate one of two contradictory beliefs) →
limited action search (recalled/generated/screened/shortlist; the globally best action may
legitimately never be considered; `actually_feasible_not_considered` is recorded) → one choice
whose decision call receives **only** the surviving material → append-only memory commit.
Every stage carries input/output hashes, provenance, branch and actor identity, a timestamp, a
model-call-or-deterministic-rule record, and failure behavior. Stage failure truncates the
branch (§ below); it never substitutes a different psychology.

## Model-family diversity (`model_families.py`)

A configured family pool with §17.1 metadata; deterministic per-(particle, actor) assignment
preserved through the branch; comparable-family failure transitions recorded; weaker families
only ever ADD adversarial coverage, never replace required strong-model particles; two
temperatures of one model are rejected as families (`FamilyIdentityError`). One-lineage runs
report `model_family_monoculture = true` on every result. Actor state stays portable across
families (the qualitative schema is provider-neutral; per-revision provenance records which
family wrote what).

## Hybrid mechanisms (`mechanism_spec.py`, `kernel_ports.py`)

Human and nonhuman mechanisms execute through one world: the canonical clock, canonical event
queue, typed `WorldState`, `StateDelta` contract, provenance, and branch-specific randomness.
`MechanismSpec` joins the operator registry, the event-type registry and the Phase-6 empirical
registry into one typed record per mechanism (kind, read/write sets, parameter sources, unified
calibration status, known limits), so fitted and broad-prior mechanisms are distinguishable at
the execution layer. Validated v1 kernels are ported with provenance-mandatory parameters
(poll-error measurement, whip-count institutions, outside-world hazard calendar, mean-field
population exposure); the full audit and accept/reject reasons live in
`artifacts/core_arch/kernel_migration_manifest.json`. External simulators attach only through
`ExternalSimulatorAdapter` records that declare version, data cutoff, units, seed behavior and
counterfactual safety — an LLM can never register executable code. There is no fake universal
market/weather/legal simulator: missing high-sensitivity mechanisms classify
`under_modeled_nonhuman_mechanism`.

## Honest statuses, truncation, and withholding (`result.py`, `truncation.py`)

`simulation_status` distinguishes `completed`, `completed_with_degradation`, `under_modeled`
(subtypes: boundary, actor, population, nonhuman_mechanism, external_process,
parameterization, structural_disagreement), `truncated`, `clarification_required`, and
`execution_failed`. `under_modeled` and `truncated` are epistemic/representational states, not
engineering exceptions; an `under_modeled` result may carry a partial distribution that is
explicitly *conditional on the represented world boundary*. Branches carry first-class statuses
(`truncated_actor_budget`, `truncated_event_budget`, `truncated_provider_failure`, …); a
truncated branch stops at its timestamp with world state, pending events and the unresolved
decision trigger preserved — it never advances further and its weight is never redistributed.
Aggregation reports total/completed/truncated weight, reasons, affected actors, and per-option
bounds under all admissible truncated completions ("truncated branch mass is unresolved
simulation, not Monte Carlo error"). Recommendations are withheld when an unresolved boundary
component, a missing mechanism, truncated mass, or single-family support could change the
winner.

## Explicit baselines (quarantined, never default)

The numerical actor system and the generic outcome prior survive only as explicitly named
arms: `execution_policy` / `integrity="numeric_baseline_explicit"` ablations, legacy artifact
readers, and offline test comparisons under `SWM_ALLOW_NUMERIC_BASELINE=1` /
`SWM_ALLOW_GENERIC_PRIOR=1` (set in `tests/conftest.py`; enforcement tests unset them and prove
the default route cannot reach either). Production code never sets these markers.

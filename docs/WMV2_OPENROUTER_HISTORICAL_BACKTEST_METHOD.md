# WMv2 OpenRouter Historical Backtest — Method (openrouter_llama31_v1)

## Scientific status: TIER_B_PROVIDER_PINNED_POST_RELEASE (not Tier A)

- **Historical model**: `meta-llama/llama-3.1-70b-instruct` — official knowledge cutoff
  2023-12-31, **public release 2024-07-23**, open-weight reference
  `meta-llama/Llama-3.1-70B-Instruct` @ HF revision `1605565b47bb9346c5515c34102e054115b4f98b`.
- **Serving pin**: OpenRouter provider **WandB**, quantization **bf16** (endpoint metadata
  2026-07-17: WandB bf16 ctx=128000; DeepInfra serves fp8 only, so the deepinfra preference does
  not qualify under the BF16/FP16 rule; Amazon Bedrock quantization unknown → rejected).
  Request: temperature 0.2, max_tokens 2400, `provider.order=[wandb]`, `allow_fallbacks=false`,
  `require_parameters=true`, `data_collection=deny`, no plugins, no aliases. Every response is
  asserted (returned model, provider, generation id, usage) and appended to a per-row provider
  audit ledger; any mismatch fails the row (`OpenRouterEnforcementError`). Retries are bounded
  and identical-endpoint only.
- **Why Tier B**: OpenRouter/WandB cannot prove the endpoint is byte-identical to the HF revision
  or was never silently updated. Tier A = rerun the identical frozen benchmark against a
  hash-verified dedicated deployment (the interface supports it).

## Temporal ordering (fail-closed, per row)

`model_release (2024-07-23T23:59:59Z) < question_open <= forecast_cutoff < resolution_time`,
with question_open proven by the market's own archived createdAt/startDate, cutoffs generated by
the frozen deterministic rule (fractions 0.15/0.35/0.55/0.75 of the unresolved lifetime), and
resolution verified scorer-side. The release-based boundary is deliberately stronger than the
knowledge-cutoff rule: the question itself did not exist when the checkpoint froze.
Instruction/preference-stage cutoffs for Llama-3.1 are not fully documented and are recorded as
`training_stage_temporal_boundary_unverified` — another reason the post-release window is the
eligibility rule.

## Benchmark composition

100 questions × 4 cutoffs = **400 expected complete production simulations**, split
chronologically 40 calibration / 20 validation / 40 rotating locked. Scale quotas frozen before
selection (≥15/15/20/15/20/15 across single-decision-maker / small-group / multi-actor /
institutional / broad-aggregate / mixed). Exclusions, cluster caps, undecided-price guards, and
the 50-example question-form file: `benchmark_versions/openrouter_llama31_v1/
question_examples_and_selection_rules.md`. Actual composition: `composition_report.json`
(generated, sealed with the vault).

## Per-row execution

The EXACT vault `raw_question` enters `simulate_world` byte-for-byte
(`sha256(question) == question_sha256` asserted and recorded; a changed question fails the row).
Inputs: historical `as_of` = cutoff, horizon = the ex-ante scheduled deadline, the pinned
period-bounded backend, the frozen evidence capsule (`prebuilt_bundle`), frozen seed 0,
walk-forward packs, ≥200 particles. The prediction is read out of the simulated worlds
(first-passage CDF at the deadline, polarity-mapped); the LLM never writes the final probability.

## Baselines (evidence parity, same pinned model)

direct single-call probability; 5-call ensemble; 3-persona observer panel; analogical (analogs →
base rate → adjust); constant 0.5; timestamp-matched market price at the cutoff (stored in the
question vault as decision-time information). Historical base rate is computed scorer-side from
calibration outcomes. The PRIMARY comparison is complete-WMv2 vs the identical model forecasting
directly from identical evidence; capability-normalized skill = 1 − Brier_WMv2 / Brier_direct.

## Metrics

Per split / causal scale / domain / horizon: Brier, log loss, AUROC, calibration slope +
intercept, ECE, accuracy@0.5, censoring-aware event-time CRPS, 50%/80% interval coverage,
failure and full-system qualification rates, leakage census, provider-consistency failures.
Paired differences vs every baseline with event-cluster bootstrap 95% CIs. No improvement claim
when the CI includes zero.

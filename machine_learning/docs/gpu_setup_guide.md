# GPU setup guide

What you need on a CUDA box to run an 8B 4-bit QLoRA fine-tune. Data prep and the smoke test do not
need a GPU (see [`colab_guide.md`](colab_guide.md) / [`persistent_gpu_server_guide.md`](persistent_gpu_server_guide.md)).

## Install training deps

```bash
# Install a CUDA build of torch matching your driver FIRST (example — pick the right index URL):
pip install torch --index-url https://download.pytorch.org/whl/cu121

# Then the pipeline requirements (training.txt pulls in transformers/peft/trl/accelerate + bitsandbytes):
pip install -r machine_learning/requirements/base.txt \
            -r machine_learning/requirements/data.txt \
            -r machine_learning/requirements/training.txt
```

`bitsandbytes` (in `training.txt`, Linux-gated) is CUDA-only and used **only** by the 4-bit QLoRA
path. If it fails to build, you can still run the CPU LoRA smoke test — comment it out for a CPU-only
environment.

## Verify CUDA

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.cuda.get_device_name(0))"
python -c "import bitsandbytes; print('bitsandbytes OK')"
```

`torch.cuda.is_available()` must be `True`. If it is `False`, the trainer runs on CPU (full-precision
LoRA), and `bnb.optim.PagedAdamW8bit` falls back to plain AdamW — fine for the smoke test, not for an
8B run. The `train run` dry-run and `run_manifest.json` both report the effective device + `cuda` flag.

## VRAM sizing

From `reports/readiness/estimated_gpu_requirements.md` (generated by `readiness check`):

| item | estimate |
|---|---|
| VRAM: 8B 4-bit QLoRA, seq 2048, bsz 1 + grad-accum 16, grad-checkpointing on | **16–24 GB** |
| fits on | 1× A100-40GB / A6000 / RTX 4090-24GB |
| base weights (4-bit) | ~5–6 GB |
| LoRA adapter (r=16) | ~40–80 MB |
| peak activation (grad-checkpointing on) | ~8–14 GB |
| disk: base + tokenizer + cache | ~20 GB |
| disk: full normalized data + manifests | ~60–120 GB external (`$SWM_DATA_ROOT`) |

The default 8B configs (`batch_size 1`, `grad_accum 16`, `max_seq_len 2048`,
`gradient_checkpointing: true`) fit comfortably on a 40 GB GPU. If VRAM-constrained, lower
`train.max_seq_len` or LoRA `lora.r`, and raise `train.grad_accum` to keep the effective batch size.

## Launch an 8B config

```bash
export SWM_DATA_ROOT=/path/to/large/volume
export HF_HOME=/path/to/hf_home
export HF_TOKEN=...                                   # if the base model is gated (e.g. Llama)

# Prereqs: datasets prepared + a view manifest built (after human approval — see licensing_guide.md).
python -m machine_learning.cli smoke run              # MUST pass first
python -m machine_learning.cli manifests build actor_choice_v1

python -m machine_learning.cli train run 8b_actor_choice            # dry-run: prints the plan
python -m machine_learning.cli train run 8b_actor_choice --launch   # real launch
```

Checkpoints are written to `$SWM_DATA_ROOT/.../runs/<config>/` and resume automatically if the run is
interrupted. The exported adapter lands in that run dir under `adapter/`. To evaluate it, use
`evaluation.model_eval.evaluate_adapter` (see [`evaluation_guide.md`](evaluation_guide.md)).
